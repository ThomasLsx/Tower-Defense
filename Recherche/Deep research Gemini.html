Les Fondations

Installation et concepts de base de SFML.

Avant de coder la logique du jeu, il est crucial de mettre en place un environnement de développement fonctionnel. Cette section vous guide pour installer SFML et comprendre le concept fondamental de la boucle de jeu, qui est le cœur de toute application interactive.

1. Installer SFML

Téléchargez la bonne version de SFML pour votre compilateur (ex: MinGW pour Code::Blocks, MSVC pour Visual Studio) et décompressez-la dans un dossier accessible.

2. Configurer le Projet

Configurez les chemins d'accès aux fichiers d'en-tête (`include`) et aux bibliothèques (`lib`) de SFML dans les options de votre projet. Liez les bibliothèques nécessaires (graphics, window, system).

3. La Boucle de Jeu

Le cœur de votre application. C'est une boucle `while` qui s'exécute tant que la fenêtre est ouverte et qui gère les événements, met à jour la logique et affiche les graphismes à chaque tour.

Architecture du Jeu

Conception Orientée Objet (POO).

Une bonne architecture est la clé d'un projet maintenable et évolutif. Penser en termes d'objets (classes) avec des responsabilités claires simplifiera grandement le développement. Voici une proposition de structure de classes de base. Cliquez sur chaque classe pour voir ses responsabilités et attributs potentiels.

Jeu

Responsabilités :
  • Classe principale, gère la boucle de jeu.
  • Contient la fenêtre, la carte, les tours, les ennemis.
  • Gère l'état du jeu (en cours, gagné, perdu).
Attributs :
  • sf::RenderWindow window
  • Carte maCarte
  • std::vector<Tour> tours
  • std::vector<Ennemi> ennemis

Carte

Responsabilités :
  • Charge et représente la carte du jeu (grille).
  • Définit les zones constructibles et le chemin.
  • Fournit les données pour le pathfinding.
Attributs :
  • std::vector<std::vector<int>> grille
  • sf::Vector2i pointDepart
  • sf::Vector2i pointArrivee

Entité (Abstraite)

Responsabilités :
  • Classe de base pour tous les objets du jeu.
  • Possède une position, un sprite.
  • Définit des méthodes virtuelles (update, draw).
Attributs :
  • sf::Vector2f position
  • sf::Sprite sprite

Projectile

Responsabilités :
  • Créé par une tour, se déplace vers un ennemi.
  • Inflige des dégâts à l'impact.
  • Est détruit après avoir touché sa cible.
Attributs :
  • int degats
  • float vitesse
  • Ennemi* cible

Tour (hérite d'Entité)

Responsabilités :
  • Peut être construite par le joueur.
  • Détecte et tire sur les ennemis à portée.
  • A un coût, une portée, une cadence de tir.
Attributs :
  • int cout
  • float portee
  • float cadenceDeTir

Ennemi (hérite d'Entité)

Responsabilités :
  • Apparaît par vagues.
  • Suit le chemin défini par le pathfinding.
  • Possède des points de vie, une vitesse.
Attributs :
  • int pointsDeVie
  • float vitesse
  • std::vector<sf::Vector2i> chemin

L'Algorithme Clé : Le Pathfinding

Comment les ennemis trouvent leur chemin.

Le cœur de l'IA de vos ennemis est leur capacité à trouver le chemin le plus court vers leur objectif. Cela se fait en modélisant votre carte comme un graphe (une grille de nœuds) et en utilisant un algorithme pour trouver le chemin optimal. Les deux algorithmes les plus courants pour cela sont Dijkstra et A* (A star).

Critère Algorithme de Dijkstra Algorithme A* (A Star)
Principe Explore tous les chemins possibles à partir du point de départ, en s'étendant uniformément dans toutes les directions. Similaire à Dijkstra, mais utilise une "heuristique" pour estimer la distance restante et prioriser les chemins qui semblent les plus prometteurs.
Performance Garanti de trouver le chemin le plus court, mais peut être lent car il explore de nombreuses zones inutiles. Généralement beaucoup plus rapide que Dijkstra car il explore de manière plus ciblée. Trouve aussi le chemin le plus court si l'heuristique est bien choisie.
Recommandation Bon pour commencer et comprendre les bases de la recherche de chemin. Fortement recommandé pour ce projet. C'est la norme dans le développement de jeux pour sa performance.

Plan de Travail Suggéré

Une approche itérative pour construire votre jeu pas à pas.

Ne tentez pas de tout faire en même temps. Progressez par étapes, en vous assurant que chaque fonctionnalité est stable avant de passer à la suivante. Cela facilitera la rédaction de vos rapports bi-hebdomadaires et le débogage.

Phase 1 : Les Bases

  • Créer le projet et configurer SFML.
  • Ouvrir une fenêtre et implémenter la boucle de jeu.
  • Dessiner la grille/carte de base à l'écran.

Phase 2 : Gameplay Initial

  • Implémenter les classes `Tour` et `Ennemi`.
  • Permettre au joueur de placer des tours avec la souris.
  • Faire apparaître un ennemi qui se déplace en ligne droite.

Phase 3 : L'IA

  • Implémenter l'algorithme de pathfinding A*.
  • Faire en sorte que l'ennemi suive le chemin trouvé.
  • Gérer le recalcul du chemin si le joueur place une tour.

Phase 4 : Interaction

  • Les tours détectent les ennemis à portée et tirent.
  • Gérer les collisions projectiles/ennemis et les points de vie.
  • Implémenter les vagues d'ennemis.

Phase 5 : Finitions

  • Ajouter une interface utilisateur (points de vie, argent...).
  • Gérer les conditions de victoire et de défaite.
  • Nettoyer le code et ajouter des commentaires.
